Quick Start

  1. Click + Add Stem in the left panel and pick an audio file from your media/audio/ folder.
  2. An effect is assigned automatically. Use the dropdown in the stem row to change it.
  3. Click the stem row to select it — its parameters appear in the right panel.
  4. Hit Space to play. Adjust Drive and other params while it plays.
  5. Add more stems to layer effects. Each stem has its own effect, blend mode, and volume.
💡 Tip: Duplicate a stem with Shift+D, then assign a different effect and blend mode. Stacking the same audio through two effects with different blends gives rich composite visuals.

Interface Overview

Left Panel — Stems

Each loaded audio file is a stem. Stems are composited top-to-bottom: the first stem in the list is drawn on top. Drag the ⠿ handle to reorder.

Controls per stem: effect dropdown, mute (M), hide (👁), blend mode, brightness/opacity slider, remove.

Canvas Area — Viewport

All effects render here, composited by z-order. The gear icon (bottom-left) opens the Audio Feature Meters. The expand icon (top-right) toggles fullscreen.

Right Panel — Parameters

Shows sliders for the currently-selected stem's effect. Parameters auto-save to your session. Double-click any value number to type a precise value.

Timeline & Transport

The timeline shows playhead position for all stems (they play in sync). Click or drag to scrub. Transport buttons: skip to start, play/pause, skip to end.

Keyboard Shortcuts

Space Play / Pause all stems F11 Toggle fullscreen (hides all UI) Shift + D Duplicate selected stem (copy + same audio, different effect)

Stems

A stem is one audio file + one visual effect, playing in sync. You can load as many as you like. Each has its own analyser and AudioFeatureBus — so two stems respond to their own audio, not a global mix.

Stems are layered like Photoshop layers. The top stem in the left panel appears on top in the canvas. Use the drag handle (⠿) to reorder.

Controls per stem

  • Effect dropdown — switch visuals without restarting audio. The AudioFeatureBus state is preserved, so adaptive calibration doesn't reset.
  • M (Mute) — silence the audio; the visual effect keeps running from the analyser.
  • 👁 (Hide) — skip rendering this effect this frame, but audio still plays and analyses.
  • Blend — CSS mix-blend-mode: Normal, Add (additive glow), Screen, Multiply, Difference.
  • Brightness slider — opacity multiplier. Scales the blend mode's base opacity so you can fade any effect to invisible.

Symmetry Controls

The symmetry bar (bottom-right of canvas) applies kaleidoscope transformations to the final composited output in real time.

  • Horizontal — mirror left↔right
  • Vertical — mirror top↔bottom
  • Radial — N-fold rotational symmetry. Use ▲▼ to change N (2–30). Combines with H/V mirrors.
💡 Radial 6 + Horizontal makes every effect look like a mandala. Try it with Butterchurn or the Plasma shader.

Adding your own shaders

Drop any .glsl file into the shaders/ folder and restart the server. It appears in the effect dropdown under Shadertoy-*. The shader must define void mainImage(out vec4 fragColor, in vec2 fragCoord). All standard Shadertoy uniforms plus all AudioFeatureBus uniforms are available.

Parameters Panel

Selecting a stem shows its effect's parameters in the right panel. Universal parameters (Drive, Hue) appear in the highlighted box at the top — they have the same meaning across all shaders.

Universal params

  • Drive (0–2) — overall audio reactivity multiplier. 0 = completely static, 2 = maximum reactivity.
  • Hue (0–360°) — global hue rotation applied to the final output.

Double-click to type

Any numeric value in the param panel can be double-clicked to type an exact number. Press Enter or Tab to confirm, Escape to cancel.

Effect Presets

Effects that ship presets (Grid Flux, and any future effect exposing a preset API) all use the same PRESET section — it renders first in the Params panel, above Common, with a collapsible PRESET header.

  • Dropdown — factory presets plus your saved ones (★-prefixed).
  • 💾 Save — type a name and save the current settings as a user preset. It appears in the list immediately.
  • 🗑 Delete — removes the selected user preset (factory presets can't be deleted).
💡 Loading a preset then tweaking sliders shows "— custom —": your tweaks are never overwritten until you pick another preset.

3D Effects

Some effects (Grid Flux, Gaussian Splats, Plexus, Lava Lamp 2…) render true 3D scenes through a shared Three.js host. They all share:

  • Camera section — Distance, Elevation, FOV, Orbit Speed. The camera slowly orbits the scene; Orbit Speed 0 holds it still.
  • Common 3D params — Speed (time-rate of the whole effect, pre-modulation), Object Scale, position offsets.
  • Depth output — 3D scenes feed the Depth of Field FSPE a real depth buffer: add the DOF post-effect and focus/blur happen by true scene depth.

Grid Flux

A neon dot-wave landscape: one to four layered grids, each with its own deform mode (Sine, Vortex, Ripple Rings, Noise, EQ Terrain, Waveform, Combined) and render mode (Dots, Ribbons, Tubes, Contours).

  • Drag to rotate — hold the left mouse button on the viewport and drag: horizontal spins (Rotate Y), vertical tilts (Tilt X). The sliders sync on release, and the rotation saves with the scene.
  • Ripple Rings + kicks — the default Mod route wires the kick detector to Ring Impulse: every kick spawns a travelling ring. Stiffness absorbs ring energy over distance — max it and rings die before the edge.
  • EQ Terrain / Waveform — scrolling spectrogram landscape / oscilloscope history driven by this lane's audio.
  • Scene Lights — up to 3 lights (Fixed / Orbit / CrestRider — rides the tallest wave crest). Very high intensity auto-blooms.
  • Presets — 11 factory presets; 🥁 = drum-wired, 🎤 = vocal-wired, plus your own via 💾 Save.

Mods Panel — Audio Modulation

The Mods tab (right panel) routes live audio features to any effect parameter. Each route: a source (kick, snare, bass, mids, highs, level, onset, vocal note, syllable…), a target parameter, an amount, and smoothing.

  • Cyan targets — recommended destinations for that effect: parameters that animate well (frequencies, hues, impulses), not structural ones.
  • Base + delta — modulation adds on top of the slider's value. Move the slider and the modulation rides the new base.
  • Smooth — low-pass only; use punchy ALGO sources (kick/onset) for transients rather than smoothing a continuous source.
  • Listen To — visual-only lanes (video layers) pick ONE audio stem to listen to, so their Mods fire on that stem's hits, not a master mush.

LFO Rack

The LFO bar (below the stage) hosts free-running oscillators you wire to parameters by dragging an LFO's handle onto a slider. LFOs are absolute: the wave sweeps the parameter across its range around the centre. Shapes, rate (sync to the ♩ BPM readout), amplitude, and a range editor per route.

FSPE — Full-Screen Post Effects

The FSPE rack applies post-processing to a single lane (POST FX on a stem) or the final composite (MASTER bus): Bloom, Levels, Chromatic Aberration, Depth of Field, Pixel Dots, Echo Smear, Feedback Zoom, Liquid Metal, Halftone, Fisheye, Contour Lines, Depth Fog and more — chain and reorder freely.

  • Drive — most FSPEs have audio drive; Mods can target FSPE params too (cyan = recommended).
  • Depth of Field — with a 3D effect on a lane, DOF blurs by real scene depth. Focus distance and range are in scene units; bokeh size sets the blur disc.

MIDI Mapping

Any parameter can be MIDI-mapped: right-click a slider → MIDI Learn, move a controller. Mappings write the parameter's base (so LFOs and Mods compose on top), survive scene switches, and save with the session. Mapped params show a small glyph.

Import / Fetch

The ⬇ Fetch button in the header opens the import modal. It brings outside shaders and MilkDrop presets into Vibronaut, adapts them to run here, and — critically — reaches inside them to expose meaningful sliders (Deep Params). Everything is sidecar-driven: importing writes files, then the normal loader picks them up — there is no manual registry step.

💡 The golden rule: the untouched original is always copied to shaders/SOURCE/… first and never edited again. Everything the pipeline adapts happens on a separate working copy. If an import goes wrong, the pristine source is safe.

GLSL shaders — the exact steps

  1. Load the raw code (URLs go through the server's proxy-url to avoid CORS).
  2. Route by type. Non-.milk → the GLSL path (below). Extension decides — a .milk is never imported as .glsl.
  3. Analyse for Deep Params (POST /api/shaders/analyze). If an Anthropic key is present, an LLM authors a refine spec — the exact const → uniform rewrites plus labelled params that reach into the shader's math. No key → skipped (generic knobs only).
  4. Adapt + wire (wireImportedShader): normalise the code (void main()→mainImage, strip #version/precision/#ifdef GL_ES, remap GLSL-Sandbox time/resolution/mouse), rename the shader's mainImage→mainImageInner, and append a host wrapper that adds Common (Drive, Hue) plus 3–8 generic controls (scale, brightness, contrast, saturation, vignette). If a refine spec exists, its constants become real u_<key> uniforms instead. A channel-select param is added for each iChannelN the shader samples.
  5. Back up + write (POST /api/shaders/import, saveSource:true): the pristine original → shaders/SOURCE/<category>/; the wired working copy → shaders/<category>/<name>.glsl; a .json sidecar holds the params. Category ∈ shadertoys / glsl / other (never vibronaut or butterchurn for third-party).
  6. Compile-check offscreen (WebGL2). A shader that fails to compile is still saved, flagged status:"error" in its sidecar, and kept out of the "broken" state in the dropdown.
  7. Speed auto-calibration (calibrateAnimSpeed): render a few frames offscreen, measure the median frame-to-frame motion, map inversely to a calm animSpeed, and persist it (POST /api/shaders/set-speed) so hyperactive shaders don't strobe.
  8. Register: re-run loadShaderPresets() — it scans the sidecars, injects COMMON_PARAMS, validates every param against a real u_<key> uniform (drops any that don't exist), and registers EFFECTS['preset_<name>']. The effect now appears in the dropdown / Effects browser.

MilkDrop .milk presets — the exact steps

A .milk is not a standalone effect. It becomes a preset inside the Butterchurn effect, shown in the Butterchurn tab of the Effects browser (not the plain stem dropdown).

  1. Load the raw preset and read its PSVERSION_WARP/COMP/PSVERSION. If any is ≥ 4 it's a MilkDrop 3 pure-shader preset → the MilkDrop 3 → GLSL path instead (Butterchurn can't render those).
  2. Convert in-browser: MilkDrop's EEL equations are transpiled to JS and its HLSL warp/comp shaders to GLSL by jberg's converter (lazy-loaded, ~2 MB). Output = a converted JSON with init_eqs_str / frame_eqs_str / pixel_eqs_str (compiled JS) plus warp / comp (GLSL) and baseVals.
  3. Analyse for Deep Params (POST /api/shaders/analyze-milk with the converted JSON) → a milkParams spec (see Deep Params). LLM if a key is set, otherwise a deterministic heuristic.
  4. Back up + write (POST /api/shaders/milk-import): pristine .milk → shaders/butterchurn/SOURCE/; working .milk → shaders/butterchurn/; the converted JSON → shaders/butterchurn/converted/<name>.json; a meta sidecar (with the validated milkParams) → shaders/butterchurn/meta/<name>.json. Skip-not-overwrite.
  5. Register: loadMilkdropPresets() prefers the converted/ JSON and adds the preset (★-prefixed) to the Butterchurn tab. Thumbnails auto-generate.
💡 Equation/coefficient tweaks in the converted JSON take effect with no re-convert; only edits to the warp/comp GLSL need the browser converter re-run (tools/butterchurn-convert/convert.html → "Convert all").

MilkDrop 3 (pure-shader) presets

A preset whose whole image is generated in the warp shader (PSVERSION ≥ 4 — raymarches like MilkDrop2077) can't render through Butterchurn's mesh pipeline. Instead the warp shader — a self-contained Shadertoy raymarch — is ported to a native GLSL effect (portMilkdrop3), imported as category:'glsl' with the raw .milk kept as its SOURCE, and demarcated in the info panel as "MilkDrop 3 → GLSL". If the port fails it falls back to the ordinary Butterchurn import (never a silent black effect).

Deep Params — reaching inside the shader

Deep Params are Vibronaut's term for a control that reaches inside a shader/.milk/.glsl and meaningfully changes the final look — as opposed to the shallow Common knobs (Drive, Hue) or generic output post-processing. They appear under a "Deep Params" heading in the right panel.

GLSL — refine spec (const → uniform)

Generic wrapper knobs are shallow. A meaningful param (e.g. "Sphere Count", "Fold Iterations") requires rewriting a hard-coded constant into a u_<key> uniform. The refine spec lists those const → uniform replacements plus the param definitions; safe targets are scalar / colour constants and loop-bound counts (kept as a fixed MAX + if (i >= u_count) break;).

.milk — milkParams literal edits

Each Deep Param maps a slider to a literal find/replace edit of one of the preset's own strings — pixel_eqs_str / frame_eqs_str (JS) or warp / comp (GLSL). find is an exact verbatim substring; replace is that same substring with one numeric literal swapped for ${v} — so an edit can only ever change that one number, never corrupt the preset. On change the preset is reloaded with the edit applied. A param may instead carry baseVal (no edit) to override a whitelisted MilkDrop baseVal like decay.

Which literals get exposed (priority order)
  1. Trails — the feedback-persistence constant in the decay equation (e.g. a['decay']=((0.997+…). The single strongest, clearest lever: sparse wisps ↔ shapes that fill the frame. (Exposed as an equation edit because the running effect can't drive the raw decay baseVal.)
  2. comp/warp GLSL constants when the preset HAS a fragment shader — the dominant look (flash, glow, shape, symmetry) usually lives there.
  3. Warp-mesh geometry coefficients for classic presets with NO shader — the *N magnitude multipliers on the mesh-warp outputs zoom / rot / warp / cx / cy / sx / sy / dx / dy (e.g. …*a['q6'])*0.3 = rotation strength). These are the strong motion/shape knobs.
  4. Generic equation literals as last-resort filler.

Never a literal that only feeds a cosmetic output — ob_*/ib_* (border colours) or wave_* (waveform colour/position). Those are near-invisible ("does nothing"). Cap: 8 params.

Fix Up Deep Params (Options → Library)

As the extraction improves, the Fix Up Deep Params button re-runs it across every .milk preset — filling in ones with no Deep Params and upgrading weak old ones, all from the existing converted JSON (no re-import). Preview (dry run) shows what would change; Apply writes it (prior meta backed up as .bak).

Hand-tuned presets are never touched. A preset marked curated (its Deep Params were manually tuned + visually validated) is skipped by default — the fix-up only regenerates candidates; it does not visually validate them. Each reprocess stamps a date, so the ⓘ info panel shows both Added (original ingestion) and Deep Params — refreshed (the latest fix-up).

⭐ Quality gate (non-negotiable): the pipeline only produces candidate params. Every exposed slider is verified to visibly change the look (render at default vs. a moved value) before it ships — an inert param is dropped or replaced. And a do-no-harm invariant holds: a default load renders byte-identical to the untouched preset (an edit only applies once the slider is moved off its authored value).

Audio Feature Bus

Every stem has its own AudioFeatureBus — a signal processor that runs 60 fps and computes 13 named musical features from the raw FFT. These signals drive all visual reactivity.

Auto-classification

The bus reads the stem filename and automatically tunes its sensitivity for the stem type. A file named drums.flac gets tighter onset detection and faster release. A file named pad.wav gets a long sustain envelope and a high onset threshold. Types: drums, bass, vocals, pad, melodic, fx.

If the name gives no signal, the bus listens to the audio for ~200 active frames and classifies by spectral content. Check the browser console to see what it chose.

Signal reference

SignalTypeDescription
bass adaptive Sub-bass energy (0–250 Hz). Self-calibrates to the stem's peak — a quiet bass stem still reads 0.8 on its loudest notes.
mid adaptive Midrange energy (250 Hz–3 kHz). Adaptive-normalised per stem.
high adaptive Treble energy (3–10 kHz). Adaptive-normalised per stem.
energy adaptive Full-spectrum RMS. Adaptive-normalised — self-calibrates to the stem's overall loudness.
brightness adaptive Spectral centroid: 0 = bass-heavy, 1 = treble-heavy. Follows the timbral character of the sound.
flux adaptive Spectral flux — how much the spectrum changed since the last frame. High on attacks and fills.
onset pulse Global transient pulse. Fires 1.0 on any detected hit, then linearly decays to 0 over ~250 ms. Cooldown prevents burst-firing.
kick pulse Bass-band onset pulse. Fires on sudden bass energy jumps (kick drums, bass plucks). Same decay shape as onset.
snare pulse Mid+high blend onset pulse. Fires on broad-spectrum transients (snare cracks, hand claps, piano keys).
hat pulse High-band onset pulse. Fires on hi-hat taps and high-frequency clicks. Very sensitive — fast cooldown.
level raw Instantaneous raw RMS across all bins. Not adaptive — 0 = silence, ~0.3 = loud. Comparable across stems.
rms raw VU-meter envelope of level (30 ms attack, ~500 ms release). Follows loudness shape. Comparable across stems.
note raw Sustained-note activity. Very slow release (0.35–2.5 s depending on stem type) + noise gate. High during held notes, zero during rests.

Adaptive vs raw signals

Adaptive signals (bass, mid, high, energy, brightness, flux) self-calibrate to each stem's dynamic range over time. A quiet stem still reads 0–1 fully. Use these for per-stem visual reactivity.

Raw signals (level, rms, note) are absolute and comparable across stems. A quiet vocals stem reads proportionally lower than a loud drums stem. Use these for mix-level decisions or cross-stem comparisons.

Audio Feature Meters

Click the ⚙ gear icon at the bottom-left of the canvas to open the Feature Meters overlay. It shows live values for all 13 signals of the currently-selected stem, updating 60 fps.

  • The coloured bars show signal 0–1 normalised width
  • The number on the right shows the exact value
  • Bars update with a 40 ms smoothed transition so they don't flicker

Use the meters to verify the stem is classified correctly and that kick/snare/hat are firing on the right hits. Check the browser console for the classification log line.

Drive & Sensitivity

Drive is a global sensitivity gate present on every effect. It scales how strongly audio signals affect the visuals.

  • Drive = 0 — completely static. The visual plays but doesn't react to audio at all.
  • Drive = 1 — moderate reactivity (the default sweet spot).
  • Drive = 2 — maximum. The visual reacts very aggressively — large size changes, bright flashes on beats, strong camera movement.
💡 For a calm ambient look: load a pad or ambient audio stem, set Drive = 0.2–0.4. The visual breathes slowly without hard hits.

Save & Load

Your session (stems, effects, parameters, blend modes, symmetry state) is auto-saved to localStorage every 300 ms and restored on next page load.

File sessions

Use the 💾 Save and 📂 Load buttons in the header to save/load named session files. Files are stored in sessions/ on the server. This lets you save multiple presets (e.g. "drums-only.json", "full-band.json") and switch between them.

Scenes

A Scene captures the whole stack — stems, effects, params, Mods, FSPE chains — as a card in the Scenes strip. Click a card (or use ← / →) to switch live: switches are seamless (the outgoing frame holds until the incoming scene's first clean frame).

  • + Scene — capture the current stack. The active scene auto-updates as you tweak.
  • Ghost Stems — thumbnails of stems from other scenes; click to pull one into the current scene.
  • Timeline cues — add scene-switch ticks on the transport timeline for scripted set changes.

Lyrics Overlay

The Lyrics button (top bar) renders timed lyrics on the MASTER bus for processed songs that have them. Style, size, position and slow-fade are configurable; it composites over everything and records into takes.

AI Studio — Offline Renders

Offline Render (top bar) queues AI-generated loopable clips into media/ai/video/, rendered locally through ComfyUI. Pick a STYLE first, then a look:

  • Abstract Loop — seamless abstract loops, 13 looks, hallucination dial.
  • LongForm — minutes-long evolving pieces (nature, weather, architecture…) with story beats.
  • Context Smash — mask-driven style morphs (bring a mask video, get style-locked motion).
  • Infinite Zoom — endless dream-zoom dives.

The AI Render Monitor (top bar) shows queue, progress and finished clips; the gallery grades each clip's loop-seam, flicker and motion (PASS / WARN badges). Renders appear in the media browser's AI Renders tab — click one to add it as a video layer.

Upscale to 4K

Any render can be upscaled from the gallery: choose an engine Type, a target (1440p / 4K) and a Detail Quality tier. Upscaled clips land in media/ai/video/upscale/.

AI Feed — Live AI Frames

The AI toggle streams the live composite through a local img2img model and blends the result back — a continuously re-painted version of your visuals. Model, LoRA, prompt sets (per-effect word libraries), strength and cadence live in the AI window. Needs the local bridge running; everything stays on this machine.

Process Song — Stem Separation

Process Song splits any audio file into 6 stems (vocals, drums, bass, guitar, piano, other) with Demucs, offline, into media/music/<Song>/ — plus a source-master reference for Master Takes and lyrics timing. Load the stems as separate lanes and give each its own visual.

Record to Disk

Two recorders live in the transport row. Both capture the final composited output — every layer, blend, symmetry and post-FX — straight from the compositor (no tab-sharing, no browser prompts) at full frame-rate, and save to media/OUTPUT/.

StemRec

Records the visuals with the live stem mix — exactly what you hear: muted stems stay out, fader moves are captured. The everyday take.

MASTER Take

Records the visuals with the original source audio (the full-quality master the stems were split from), not the stem mix — for final videos. While a MASTER take runs, mouse and keyboard are locked out so nothing disturbs the take; your MIDI controller keeps working — perform the take on MIDI. Esc ends it.

  • Scene switches during a take are seamless — switch freely mid-recording.
  • Takes auto-stop at the end of the track (or stop them manually).
  • Recording runs on the GPU encoder; frame rate stays at ~60fps while recording.

Writing Shaders

Vibronaut shaders are standard Shadertoy-compatible GLSL (WebGL 2 / GLSL ES 3.00). Write a mainImage() function and drop the file in shaders/. Reload the page.

To add parameters: declare uniform float u_myParam; in your shader, then add an entry to PRESET_CONFIGS in effects/effectRegistry.js. The param panel renders it automatically.

💡 Use #define gl_FragColor outColor (already included) to paste Shadertoy shaders directly without modification. The iChanEnvUV(vec3 dir) helper converts a 3D direction to equirectangular UV for sampling image textures as environment maps.

Every shader automatically receives these uniforms. No declaration needed — they're in the preamble injected by glslBase.js.

iChannel0 layout

iChannel0 is a 512×2 texture. Row 0 (y=0.25) = FFT magnitudes 0–1. Row 1 (y=0.75) = waveform. Sample with texture(iChannel0, vec2(x, 0.25)).r where x = normalised bin index.

iChanEnvUV helper

Use iChanEnvUV(vec3 direction) to convert a 3D reflection or refraction vector to a UV coordinate for sampling an equirectangular panorama stored in iChannel1–3. This is how environment maps work in the crystal ball and tunnel shaders.
